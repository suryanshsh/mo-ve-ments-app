These tasks simulate\nrealistic DevOps workflows where problems cascade through the development pipeline.\nEach pipeline task follows a four-stage sequence: Stage 1: Build. Starting with a monitoring-ready\nrepository, we inject build configuration errors (e.g., missing dependencies, incorrect paths) that\nprevent compilation. Agents must diagnose and fix these issues to establish a working baseline.\nStage 2: Monitoring and Detection. Once the build succeeds, agents deploy and monitor the\nrunning system, which contains a latent performance or resource issue from our monitoring task\nset (3.3). Agents must use system tools to identify anomalies without prior knowledge of the issue\ntype. Stage 3: Issue Resolving. Based on monitoring findings, agents implement code-level fixes\naddressing the root cause. Stage 4: Validation and Testing. Our goal in DevOps-Gym is to cover some core workflows. For each candidate repository, we sample 20 unique\ncode snippets from CI/CD configurations, build files, and test files, which are domain-specific\nartifacts less likely to appear in general pre-training data. Following the prefix completion ap-\nproach (Carlini et al., 2021; Staab et al., 2023), we first randomly select a start point, then extract a\n50-token prefix from each snippet and prompt the model to generate a continuation. We then assess\ncontamination by comparing the model’s 50-token prediction a with the ground truth suffix g from\nthe original code.\nWe measure similarity using five complementary metrics: (1) normalized Levenshtein distance ratio\ns (Lcvenshtcin, 1966), (2) consecutive matching token count from the prefix t, (3) position of the\nfirst token mismatch p, (4) longest common substring length l measured in tokens (Gusfield, 1997),\nand (5) sentence-level BLEU score b (Papineni et al., 2002). A snippet is classified as high-risk if\nany metric exceeds its respective threshold: s > 0.7, t > 30, p > 30, l > 30, or b > 0.5. These\nthresholds are calibrated to identify cases where the model demonstrates suspiciously accurate re-\nproduction, indicating potential memorization.
